# iOS Build Guide for EJEMMA Wallet

## Prerequisites

1. **macOS machine** with Xcode 15+ installed
2. **Apple Developer account** (
99/year) or a free Apple ID for local device testing
3. **Flutter stable channel** installed on the Mac
4. **CocoaPods** installed (`sudo gem install cocoapods`)
5. **Fastlane** (optional but recommended) (`brew install fastlane`)

## One-Time Project Generation (Linux cannot do this)

From a Mac, inside the repo:

```bash
cd agbara-wallet-flutter
flutter config --enable-ios
flutter create . --platforms=ios,android,web
```

This regenerates the `ios/` directory with a proper Xcode workspace.

## Bundle ID & Signing

Open `ios/Runner.xcworkspace` in Xcode:

1. Select **Runner** in the project navigator
2. Go to **Signing & Capabilities**
3. Set **Team** to your Apple Developer team
4. Set **Bundle Identifier** to `info.ejemma.wallet`
5. Xcode will automatically create/provision a signing profile

## Build IPA (for TestFlight / App Store)

```bash
flutter build ipa --release
```

This creates an `.xcarchive` and exports an IPA to:
`build/ios/ipa/ejemma_wallet.ipa`

## Build IPA via Command Line with ExportOptions.plist

An `ExportOptions.plist` has been prepared at `ios/ExportOptions.plist`. Replace `REPLACE_WITH_YOUR_TEAM_ID` with your Apple Team ID, then:

```bash
flutter build ios --release

cd ios
xcodebuild -workspace Runner.xcworkspace \
  -scheme Runner \
  -sdk iphoneos \
  -configuration Release \
  archive -archivePath build/Runner.xcarchive

xcodebuild -exportArchive \
  -archivePath build/Runner.xcarchive \
  -exportPath build/Runner.ipa \
  -exportOptionsPlist ExportOptions.plist
```

The IPA will be in `ios/build/Runner.ipa/`.

## What the App Needs from Apple

| Requirement | Why |
|---|---|
| Apple Developer Program | Required to upload to App Store / TestFlight |
| App Store Connect record | Bundle ID registration, screenshots, metadata |
| Provisioning profile | Xcode generates this after team selection |
| iOS Privacy manifest | Generated by Flutter; verify `NSPrivacyCollectedDataTypes` |

## Known iOS Considerations

- **Biometric auth** uses `local_auth` with iOS messages already configured in `lib/services/auth_service.dart`
- **Camera/QR scan** uses `mobile_scanner`; iOS camera permission string is in `Info.plist` after generation
- **Secure storage** uses `flutter_secure_storage` with iOS Keychain
- **Custom URL schemes**: `bitcoin:`, `liquidnetwork:` are handled by the app

## No macOS Available?

If you don't have a Mac, use **GitHub Actions macOS runner**:

```yaml
# .github/workflows/ios.yml
name: Build iOS
on: push
jobs:
  build:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v4
      - uses: subosito/flutter-action@v2
        with:
          flutter-version: '3.22.0'
      - run: flutter pub get
      - run: flutter build ios --release --no-codesign
```

For App Store distribution, you must add `MATCH_PASSWORD`, `FASTLANE_PASSWORD`, and Apple Team ID as GitHub secrets.
